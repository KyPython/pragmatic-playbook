# First Cold Email Campaign Checklist

Complete checklist to launch your first 30-50 prospecting emails this week.

## Pre-Launch (Day Before)

### Apollo Setup
- [ ] Apollo account created and verified
- [ ] Workspace configured
- [ ] Domain verified (foundersinfra.com)
- [ ] SPF/DKIM/DMARC records added
- [ ] Sender email set: founders@foundersinfra.com
- [ ] Email tracking enabled

### List Preparation
- [ ] ICP list created in Apollo
- [ ] List size: 300-800 contacts (or more)
- [ ] Filters applied (see `01-consulting-icp-profile.md`)
- [ ] Data enriched (company info, funding, etc.)
- [ ] Invalid emails removed
- [ ] First 50 contacts selected for test batch

### Sequence Setup
- [ ] Email sequence created: "Consulting Outreach"
- [ ] Email 1 added (Day 0)
- [ ] Email 2 added (Day 3)
- [ ] Email 3 added (Day 7)
- [ ] Personalization tokens tested
- [ ] Automation rules configured
- [ ] Test email sent to yourself
- [ ] Test email verified (received, formatted correctly)

### Content Review
- [ ] Subject lines reviewed
- [ ] Email content reviewed
- [ ] CTAs clear and actionable
- [ ] Personalization working
- [ ] No typos or errors
- [ ] Links tested and working

## Launch Day

### Final Checks
- [ ] 30-50 contacts selected
- [ ] Sequence assigned to contacts
- [ ] Sender email confirmed: founders@foundersinfra.com
- [ ] Send time set (best: Tuesday-Thursday, 9 AM - 2 PM)
- [ ] Tracking dashboard ready
- [ ] Response tracking template ready (see `05-email-tracking-template.md`)

### Launch
- [ ] Start sequence
- [ ] Confirm first batch sent
- [ ] Note launch time
- [ ] Set reminder to check in 24 hours

### Immediate Post-Launch
- [ ] Check deliverability (should be > 95%)
- [ ] Verify emails are sending
- [ ] Check for any immediate bounces
- [ ] Set up daily monitoring schedule

## Day 1 (24 Hours After Launch)

### Morning Check (9 AM)
- [ ] Check Apollo dashboard
- [ ] Review metrics:
  - [ ] Emails sent: ___
  - [ ] Delivered: ___%
  - [ ] Opens: ___ (___%)
  - [ ] Replies: ___
  - [ ] Bounces: ___

### Response Handling
- [ ] Check inbox for replies
- [ ] Log all replies in tracking template
- [ ] Categorize responses:
  - [ ] Interested
  - [ ] Qualified
  - [ ] Not interested
  - [ ] Out of office
  - [ ] Other

### Follow-ups
- [ ] Respond to interested contacts (within 4 hours)
- [ ] Send calendar link to qualified leads
- [ ] Thank "not interested" replies
- [ ] Schedule follow-up for OOO replies

### Documentation
- [ ] Update tracking template
- [ ] Note any patterns (best send time, subject line performance)
- [ ] Document lessons learned

## Response Templates Ready

Have these ready to copy/paste:

### Interested Response
```
Hi [Name],

Thanks for the reply! I'd love to learn more about [Company]'s infrastructure challenges.

Quick question: What's your biggest pain point right now?
- Deployment speed?
- Technical debt?
- Team scaling?

I can help with a free 15-minute call to discuss. Would [Day] at [Time] work?

Best,
[Your Name]
```

### Qualified Lead Response
```
Hi [Name],

Great to hear from you! Based on what you mentioned, I think I can help.

I've helped similar Seed/Series A startups:
- Reduce deployment time from 1.5 hours to 8 minutes
- Recover $50K+ in lost velocity
- Achieve 70% code coverage in one week

Would you be open to a 15-minute call to discuss your specific challenges?

[Calendar Link]

Best,
[Your Name]
```

### Not Interested Response
```
Hi [Name],

No problem at all - thanks for letting me know!

If you know any other Seed/Series A CTOs who might be interested, I'd appreciate an intro.

Best of luck with [Company]!

Best,
[Your Name]
```

## Quick Reference

**Target Metrics (First 50 Emails):**
- Deliverability: > 95%
- Open Rate: 30-35%
- Reply Rate: 3-5% (1-2 replies)
- Qualified Leads: 1-2
- Booked Calls: 0-1

**Best Practices:**
- Send Tuesday-Thursday
- Send 9 AM - 2 PM (recipient timezone)
- Personalize every email
- Respond to replies within 4 hours
- Track everything

**Time Investment:**
- Setup: 1 hour
- Daily monitoring: 15 minutes
- Follow-ups: 30 minutes per reply
